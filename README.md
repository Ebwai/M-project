# M-project · 第三阶段：行为同步（A-project）

> 工具 & 应用场景工作流 · 社交行为 + 邮箱内容自动分发

本仓库用于承载「第三阶段：行为同步（A-project）」中，与 **社交行为自动化** 和 **邮箱内容智能处理** 相关的工作流与说明文档。

目标可以概括为你最初的一句话：

> 「社交行为：Bilibili、小红书、抖音、微信公众号的个性化风格分享；代替你去阅读随时发来的邮箱，处理并推送到你常用的社交平台上。」

---

## 🎯 项目愿景

围绕上面的目标，M-project 主要想解决三件事：

1. **代替你读邮箱**  
   - 持续监听教育邮箱 / 工作邮箱的新邮件
   - 自动过滤垃圾信息，抓住真正重要的通知、活动、机会
   - 结合邮件正文 + 附件图片（如活动海报），做语义级理解

2. **把「信息」变成「可发布内容」**  
   - 从原始邮件中抽取关键信息：时间、地点、主题、截止日期、主办方等
   - 根据你的个人偏好，用不同风格做二次创作：
     - 适合 Bilibili / 抖音 的脚本结构与标题
     - 适合 小红书 的生活化分享文案
     - 适合 公众号 的长文结构化摘要

3. **多平台协同分发**  
   - 通过 n8n 等工作流工具，把同一条邮件信息拆成多种「草稿形态」
   - 未来可逐步接入：B站、小红书、抖音、微信公众号 的半自动/自动发布接口（视平台开放程度而定）

---

## 🧠 在整体系统中的位置

- **A-project（行为层）**：
  - 更偏「具体动作」和「自动化流程」，比如：
    - 轮询邮箱
    - 解析内容
    - 生成日报
    - 推送提醒
- **M-project（记忆与心智层）**：
  - 更关注「长期偏好」和「个人记忆」，例如：
    - 你长期关心哪些主题（求职 / 海外项目 / 比赛 / 奖学金等）
    - 你想要在各平台上呈现的个人风格（理性 / 松弛 / 幽默 / 技术流）
  - 未来会逐步把这些偏好融入到每一次邮件重写和内容生成中。

当前版本的 M-project，更多是为了：

- 承接 A-project 中已经成型的邮箱工作流与说明文档
- 为后续接入「记忆系统」和「多平台社交输出」提供一个清晰的项目外壳

---

## 📁 当前仓库包含内容

本仓库目前主要包含：

- 与 **EduEmail AI Agent** 相关的说明文档
- 基于 **n8n** 的邮箱工作流导出文件（JSON）
- 围绕「自动读邮箱 → 打分分类 → 提醒/日报 → 后续分发」这一闭环的核心逻辑

后续你可以逐步在此仓库中加入：

- 针对不同平台的文案模板与 Prompt 设计
- 与 M-project 记忆库对接的接口定义
- 更多实际使用过程中的调整记录

## 🚀 下一步可以做什么？

- 把现在的 EduEmail 工作流产物（重要邮件提醒、每日邮件精选日报），作为「内容源头」接入 M-project
- 为 B站 / 小红书 / 抖音 / 公众号 分别设计一版「从邮件到内容草稿」的自动生成流程
- 逐渐让「读邮件 + 选重点 + 想标题 + 写文案」这套流程，完全从你脑中迁移到 AI + 工作流系统中

当这些都连起来之后，你会得到一个：

> 从邮箱 → 个人记忆 → 多平台社交输出

的「行为同步」自动化系统，这正是 M-project 想要承载与延展的方向。
